# 動態路由文件分析系統 (Dynamic Document Analysis System)

一個支援 AI 智能分析的 OCR 文件解析系統，能夠自動識別文件內容、提取關鍵資訊，並通過 LLM 進行智慧診斷。

## 📋 專案概述

此系統提供以下核心功能：

- **智慧 OCR 辨識**：支援 PDF、PNG、JPG 等多種文件格式，採用 EasyOCR 引擎
- **動態文件分類**：根據文件內容自動歸類（財務文件、硬體設備、一般文件等）
- **簽核欄位提取**：自動識別並提取 PDF 中的簽名、蓋章、職位、姓名等簽核資訊
- **AI 診斷助理**：基於文件內容���行深度分析，回答使用者提問
- **多 LLM 支援**：整合 Together AI 與 Google Gemini，靈活選擇推理模型
- **視覺錯誤修正**：智能修正 OCR 誤認的形近字和特殊符號

## 🚀 快速開始

### 系統要求

- Python 3.8+
- 足夠的硬體資源（推薦 GPU 支援，用於 OCR 和深度學習加速）

### 安裝步驟

1. **複製專案**
   ```bash
   git clone https://github.com/enya091/OCR_System.git
   cd OCR_System/OCR
   ```

2. **安裝依賴**
   ```bash
   pip install -r requirements.txt
   ```
   
   首次運行時，EasyOCR 會自動下載預訓練模型（約 100MB）。

3. **配置 API 密鑰**
   
   複製 `.env.example` 為 `.env`：
   ```bash
   cp .env.example .env
   ```
   
   編輯 `.env` 並填入你的 API 密鑰：
   ```dotenv
   # Together AI 密鑰（獲取方式：https://www.together.ai/）
   TOGETHER_API_KEY=your_together_api_key
   
   # Google Gemini 密鑰（獲取方式：https://ai.google.dev/）
   GEMINI_API_KEY=your_gemini_api_key
   
   # 預設 LLM 提供商
   DEFAULT_PROVIDER=Together AI
   ```

### 啟動應用

```bash
streamlit run main.py
```

應用將在 `http://localhost:8501` 啟動。

## 💡 使用方式

### 基本流程

1. **上傳文件**
   - 在左側面板「智慧文件解析」選擇要分析的 PDF 或圖片檔
   - 支援格式：PDF、PNG、JPG、JPEG

2. **執行解析**
   - 點擊「執行解析並啟動動態路由」按鈕
   - 系統將自動：
     - 進行 OCR 辨識，提取文字內容
     - 針對 PDF，提取簽核欄位（職位、姓名、日期等）
     - 分析並分類文件類型

3. **查看結果**
   - **辨識結果**：展示提取到的完整文字內容
   - **簽核欄位辨識**：表格顯示自動提取的簽核資訊（職位、姓名、手寫內容、頁碼）
   - **圖片預覽**：檢視上傳檔案的預覽

4. **AI 診斷**
   - 在右側「內容診斷助理」中提問
   - 系統根據文件類型和內容進行回答
   - 對簽核流程、人名、職位、日期等問題優先使用已驗證的提取結果

### 文件分類類型

系統支援三種文件類型的特化處理：

| 類型 | 常數值 | 說明 |
|------|--------|------|
| **財務文件** | FINANCIAL | 轉帳截圖、發票、收據、報價單等金額相關文件。系統會自動修正 OCR 誤認的金錢符號（如 `$` 被誤認為 `5` 或 `S`） |
| **硬體設備** | HARDWARE | 螢幕標籤、產品銘牌、安規認證、硬體規格等。系統會忽略多國語言雜訊和安規標章 |
| **通用文件** | GENERAL | 合約、一般文章、其他無法歸類的文件。系統會修正掃描品質不佳或反光造成的錯別字 |

### 系統設定（側邊欄）

- **LLM 提供商**：選擇 "Together AI" 或 "Google Gemini"
- **API Key**：可在介面直接輸入，或從 `.env` 檔自動載入
  - 若介面輸入且 `.env` 已設定，則優先使用介面輸入
  - 若 `.env` 有效，左側會顯示「已偵測 .env API Key，可直接使用」
- **推理模型**：根據提供商選擇相應模型
  - Together AI：Llama-4-Maverick、Llama-3.3-70B 等
  - Google Gemini：gemini-2.0-flash、gemini-1.5-pro 等
- **OCR 引擎**：EasyOCR（懶載入，首次使用時初始化）
- **清除對話紀錄**：重置當前分析狀態

## 🔧 核心模組詳解

### 1. Document Parser（文件解析）

**主函式**：`parse_uploaded_file(uploaded_file, progress_callback=None)`

**功能**：
- 支援 PDF 和圖片（PNG/JPG）
- 對 PDF 自動選擇最佳提取方式：
  - 若有原生文字層（>50 字）→ 直接提取
  - 若無或文字過少 → 使用 EasyOCR 掃描
- 返回文字內容和預覽圖片

**輸入**：
- `uploaded_file`：Streamlit UploadedFile 物件
- `progress_callback`：進度回調函式

**輸出**：
- 元組 `(全文內容, 圖片清單)`

### 2. Approval Extractor（簽核欄位提取）

**主函式**：`extract_approval_entries_from_pdf_bytes(pdf_bytes)`

**核心提取邏輯**：
1. **收集文本塊**：從 PDF 每一頁提取所有文本及其座標
2. **識別職位**：匹配預定義職位關鍵字（`ROLE_KEYWORDS`）
3. **識別姓名**：用正則表達式提取 2-4 個中文字的候選名字
4. **空間配對**：根據座標近近度將職位與姓名配對
5. **日期修復**：標準化並修復日期格式
6. **備註提取**：識別簽核備註（「可」、「准」、「核」、「閱」）
7. **標準化輸出**：統一格式為繁體中文欄位名

**輸出格式**：
```json
[
  {
    "頁碼": 1,
    "職位": "主任檢察官",
    "姓名": "黃元冠",
    "手寫內容": "2025.03.16；可閱"
  }
]
```

### 3. Document Router（文件分類）

**主函式**：`classify_document(provider, api_key, model_id, text)`

**流程**：
1. 提取文本前 500 個字作為樣本
2. 用分類提示詞詢問 LLM
3. 規範化 LLM 回應
4. 返回標準化類型

**回傳值**：`"FINANCIAL"` 或 `"HARDWARE"` 或 `"GENERAL"`

### 4. LLM Factory（LLM 工廠）

**主函式**：`ask_model_text(provider, api_key, model_id, messages, max_tokens=2048, temperature=0.2)`

**參數**：
- `messages`：聊天歷史（包含系統訊息和使用者訊息）
- `max_tokens`：最大輸出長度（預設 2048）
- `temperature`：創意度（預設 0.2，較低表示更確定性）

**支援提供商**：
- `"Together AI"`
- `"Google Gemini"`

- **API Key 管理**：
  - 存儲在本機 `.env` 檔，不上傳到版本控制
  - 支援在 UI 中臨時輸入，優先於環境變數

## ⚙️ 進階配置

### 自訂文件分類規則

編輯 `prompt/rules.py`，為新的文件類型添加分類規則：

```python
ROUTER_RULES = {
    "FINANCIAL": "特別注意：此為財務文件。...",
    "HARDWARE": "特別注意：此為硬體設備。...",
    "CUSTOM": "新增自訂類型的特化指令...",
}
```

### 調整 OCR 引擎

編輯 `analysis/ocr_engine.py`（若存在）調整 EasyOCR 參數。

### 修改簽核提取邏輯

在 `analysis/approval_extractor.py` 中調整：
- `ROLE_KEYWORDS`：職位關鍵字
- `IGNORE_NAME_WORDS`：要忽略的詞語
- 座標匹配閾值（空間距離限制）

## ⚠️ 常見問題

### Q: 無法連線到 LLM 服務？
**A:** 
- 檢查網路連線和 VPN/代理設定
- 確認 API Key 有效且未過期
- 檢查本機 DNS 設定
- 嘗試切換到 Google Gemini

### Q: OCR 辨識準度不高？
**A:**
- 確保上傳的圖片清晰、對比度高、解析度 ≥300 DPI
- 推薦上傳 PDF 而非掃描圖片
- 對於表格/複雜版面，精度可能有限

### Q: 簽核欄位無法正確提取？
**A:**
- 確保 PDF 中簽核欄位格式相對標準
- 檢查職位是否在 `ROLE_KEYWORDS` 中
- 可在結果中手動驗證和修正

### Q: 首次運行很慢？
**A:**
- EasyOCR 首次使用需下載 ~100MB 模型（一次性）
- 推薦使用 GPU 加速（自動偵測 CUDA）
- 可在 Streamlit 側邊欄查看 OCR 引擎狀態

### Q: 如何離線使用？
**A:**
- 本地 OCR 支援離線使用
- LLM 分析必需網路連接
- 可修改程式碼存儲 LLM 回應進行本地緩存
